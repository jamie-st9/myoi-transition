# MyOi TRANSITION – Insight Platform MVP

**한 문장 정의**: 10년 이상 직장 경력자가 창업/전환을 고민할 때, 잘못된 선택을 사전에 막아주는 진단 플랫폼

---

## 제품 목적 및 핵심 가치 제안

### 제품 목적

MyOi TRANSITION은 **창업을 돕는 것이 아니라 잘못된 전환을 막는다**는 명확한 철학을 가집니다.

대부분의 창업/전환 플랫폼은 "할 수 있다"는 메시지를 전달하지만, 우리는 **"하면 안 되는 이유"**를 먼저 보여줍니다. 이는 직장인이 경력 전환 시 직면하는 가장 큰 리스크인 **확증 편향(Confirmation Bias)**과 **회피 기반 의사결정**을 깨뜨리기 위함입니다.

### 핵심 가치 제안

1. **Reality Check**: 투입시간, 손실허용금액, 실패감당범위를 정량화하여 현실적 기준 제시
2. **Career → Income Map**: 기존 경력을 수익으로 전환할 수 있는 실질적 경로 제안
3. **Decision Questions**: 회피를 깨는 질문으로 의사결정 품질 향상

## 핵심 기능 목록 (MVP 범위)

### 1. Landing Page
- **목적**: 제품 소개 및 사용자 유입
- **구성요소**:
  - Hero Section: "잘못된 전환을 막는 3가지 진단" 메시지
  - Social Proof: 베타 사용자 피드백 (익명 처리)
  - CTA Button: "3분 진단 시작하기"

### 2. Reality Check 입력
- **목적**: 사용자의 현실적 제약 조건 수집
- **입력 항목**:
  - 주당 투입 가능 시간 (10시간 미만 / 10-20시간 / 20시간 이상)
  - 손실 허용 금액 (500만 원 이하 / 1천만 원 / 3천만 원 이상)
  - 실패 감당 범위 (재취업 필수 / 1년 버틸 수 있음 / 3년 이상 가능)
  - 절대 불가 조건 (예: 가족 부양, 대출 상환, 건강 문제)

### 3. Career Snapshot 입력
- **목적**: 기존 경력 데이터 수집 및 분석
- **입력 방식**:
  - 이력서 텍스트 붙여넣기 (PDF 업로드는 MVP 제외)
  - Claude API로 자동 파싱: 직무, 역할, 반복 업무 추출
  - 사용자 확인 및 수정 기능

### 4. Idea 입력 (선택)
- **목적**: 기존 아이디어 검증 (있는 경우)
- **입력 항목**:
  - 아이디어 한 문장 요약
  - "아이디어 없음" 선택지 제공
- **처리 방식**:
  - 아이디어가 있을 경우 → Claude API로 취약점 분석
  - 없을 경우 → Career Snapshot 기반 아이디어 제안

### 5. 3-Card Report Output
- **목적**: 진단 결과 시각화 및 액션 가이드
- **카드 구성**:

#### Card 1: Reality Report
- **하면 안 되는 선택 TOP3**:
  - 예시: "주당 10시간으로 SaaS 개발은 불가능"
  - 예시: "500만 원으로 제조업 창업은 위험"
- **가능한 전환 방향 2개**:
  - 예시: "컨설팅으로 시작 → 수익 검증 후 확장"
  - 예시: "사이드 프로젝트로 시장 검증 먼저"

#### Card 2: Career → Income Map
- **수익 전환 가능 역할 3가지**:
  - 기존 경력 기반 즉시 수익화 가능 영역
  - 예시: "생산관리 경험 → 중소기업 공정 컨설팅"
- **새로 배워야 하는 영역 경고**:
  - 예시: "마케팅 경험 없음 → 고객 유치 어려움"

#### Card 3: Decision Questions
- **회피를 깨는 질문 5개**:
  - 예시: "1년 후 실패했다면, 그 이유는 무엇일까요?"
  - 예시: "이 아이디어로 첫 고객을 언제 확보할 수 있나요?"
- **위험한 가정 1개**:
  - 예시: "당신은 '시장이 있을 것이다'를 가정하고 있습니다"
- **7일 실험 1개**:
  - 예시: "7일 동안 잠재 고객 10명과 인터뷰하세요"

### 6. CTA: 14-Day Transition Plan
- **목적**: 유료 전환 유도 (MVP에서는 리드 수집용)
- **구성요소**:
  - "14일 전환 가이드" 신청 버튼
  - 이메일 수집 폼
  - Notion 기반 간단한 가이드 제공 (자동 발송)

## 비범위 (Out of Scope)

MVP에서는 다음 기능을 **의도적으로 제외**합니다:

1. **로그인/회원가입**: 세션 기반 임시 저장만 지원
2. **결제 시스템**: 유료 전환은 이메일 수집 후 수동 처리
3. **PDF 업로드**: 이력서는 텍스트 붙여넣기만 지원
4. **커뮤니티/포럼**: 사용자 간 상호작용 없음
5. **강의/콘텐츠**: 외부 리소스 링크만 제공
6. **성공 사례**: 베타 단계에서는 익명 피드백만 표시

## 사용자 플로우

```
1. Landing Page 방문
   ↓
2. "3분 진단 시작" CTA 클릭
   ↓
3. Reality Check 입력 (2분)
   - 투입시간, 손실금액, 실패범위, 불가조건
   ↓
4. Career Snapshot 입력 (1분)
   - 이력서 붙여넣기 → Claude API 파싱 → 확인
   ↓
5. Idea 입력 (선택, 30초)
   - 아이디어 1개 또는 "없음"
   ↓
6. 리포트 생성 중 (10초 이내)
   - Claude API 호출 → 3장 카드 생성
   ↓
7. 3-Card Report 확인
   - Reality Report
   - Career → Income Map
   - Decision Questions
   ↓
8. CTA: "14일 가이드 신청"
   ↓
9. 이메일 입력 → 가이드 발송
```

**핵심 메트릭**:
- 3단계 완료율 목표: 70% 이상
- 5단계까지 완료율: 50% 이상
- CTA 클릭률: 20% 이상

## 성공 지표 (KPI)

### Primary KPIs (핵심 지표)

1. **완료율 (Completion Rate)**
   - 측정: Landing → Report 생성까지 완료한 사용자 비율
   - 목표: 50% 이상
   - 측정 주기: 주간

2. **CTA 클릭률 (CTA Click-Through Rate)**
   - 측정: Report 확인 후 "14일 가이드" 신청 비율
   - 목표: 20% 이상
   - 측정 주기: 주간

3. **단계별 이탈률 (Drop-off Rate)**
   - 측정: Reality Check → Career Snapshot → Idea → Report 각 단계 이탈
   - 목표: 각 단계 이탈률 30% 이하
   - 측정 주기: 주간

### Secondary KPIs (보조 지표)

4. **리포트 품질 피드백**
   - 측정: Report 하단 "도움이 되었나요?" Thumbs Up/Down
   - 목표: Thumbs Up 70% 이상
   - 측정 주기: 주간

5. **평균 완료 시간**
   - 측정: Landing → Report 생성까지 평균 소요 시간
   - 목표: 5분 이내
   - 측정 주기: 주간

6. **재방문률**
   - 측정: 7일 내 재방문 사용자 비율
   - 목표: 15% 이상 (친구 추천 등)
   - 측정 주기: 주간

### Business KPIs (비즈니스 지표)

7. **이메일 수집률**
   - 측정: CTA 클릭 후 이메일 입력 완료율
   - 목표: 90% 이상
   - 측정 주기: 주간

8. **14일 가이드 오픈률**
   - 측정: 발송된 가이드 이메일 오픈율
   - 목표: 40% 이상
   - 측정 주기: 주간

## 리스크 및 대응

### 1. Claude API 응답 속도 (High Risk)

**리스크**: 리포트 생성 시 10초 이상 소요 시 사용자 이탈

**대응 방안**:
- Streaming Response 도입으로 카드별 순차 표시
- 로딩 애니메이션에 "진단 중..." 메시지 표시
- API 타임아웃 설정 (15초) 및 재시도 로직

### 2. 리포트 품질 일관성 (Medium Risk)

**리스크**: Claude API 프롬프트 품질에 따라 결과 편차 발생

**대응 방안**:
- Few-Shot Prompting으로 일관성 확보
- 베타 테스트 시 50+ 샘플 수집 및 프롬프트 개선
- 리포트 템플릿 구조화 (JSON 스키마 강제)

### 3. 사용자 입력 품질 (Medium Risk)

**리스크**: 이력서 붙여넣기 시 포맷 오류로 파싱 실패

**대응 방안**:
- 입력 가이드 제공 ("이력서 전체를 복사해 주세요")
- 파싱 실패 시 수동 입력 옵션 제공
- Claude API의 강건한 텍스트 파싱 능력 활용

### 4. 모바일 UX (Low Risk)

**리스크**: 모바일 환경에서 긴 이력서 붙여넣기 어려움

**대응 방안**:
- 모바일 우선 디자인 (Tailwind CSS 반응형)
- 입력 필드 크기 자동 조정
- "PC에서 작성 후 링크 전송" 기능 (향후)

### 5. 법적 리스크 (Low Risk)

**리스크**: 진단 결과에 대한 법적 책임 문제

**대응 방안**:
- Disclaimer 명시: "본 진단은 참고용이며 법적 책임 없음"
- 리포트 하단에 "전문가 상담 권장" 문구 추가
- 개인정보 처리방침 및 이용약관 준비

## 향후 로드맵 (Post-MVP)

### Phase 1: MVP 검증 (1-2개월)

- 완료율 50% 달성

### Phase 3: 커뮤니티 확장 (6개월+)
- 사용자 간 Q&A 포럼
- 전환 성공 사례 공유
- 멘토 매칭 서비스
